# /cm-rollback — 단계 되돌리기

## 실행 조건
- 프로젝트가 초기화되어 있어야 한다
- 사용자가 되돌릴 단계를 명시해야 한다

## 수행 절차

### 1. 되돌릴 단계 확인
사용자에게 확인:
- 어느 단계를 되돌릴 것인가?
- 해당 단계의 산출물을 삭제할 것인가, 보관할 것인가?

### 2. 영향 범위 확인
되돌리는 단계(M) 이후의 모든 단계(M+1~12) 상태를 확인:
- 이후 단계가 이미 진행되었다면 경고

```
⚠️ 경고: {M}단계를 되돌리면 이후 단계에 영향이 있습니다.

현재 상태:
  {M}단계: ✅ 완료
  {M+1}단계: 🔄 진행 중
  
이후 단계도 함께 되돌릴까요?
  A) {M}단계만 되돌리기
  B) {M}단계 이후 전부 되돌리기
```

### 3. 되돌리기 실행

**산출물 삭제 시:**
- 해당 단계 폴더의 산출물 파일 삭제 (FEEDBACK.md 포함)
- FEATURE-TRACKING.md에서 해당 단계 컬럼 전체 ⬜로 초기화

**산출물 보관 시:**
- 해당 단계 폴더에 `_backup/` 하위 폴더 생성
- 기존 산출물을 `_backup/`으로 이동

### 4. DASHBOARD.md 갱신
되돌린 단계의 상태를 `⏳ 대기`로 변경.
STATE.md가 있으면 동기화.

### 5. 완료 보고

```
🔙 {단계명} 단계를 되돌렸습니다.

{삭제/보관} 처리된 파일:
  - {파일 목록}

DASHBOARD.md 갱신 완료.

👉 해당 단계를 다시 시작하려면 /{CM 스킬}을 실행하세요.
```
