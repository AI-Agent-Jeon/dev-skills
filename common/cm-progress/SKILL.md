# /cm-progress — 진행 상태 확인

## 수행 절차

### 1. 프로젝트 존재 여부 확인

CLAUDE.md 파일이 현재 디렉토리에 있는지 확인한다.

**없는 경우:**
```
프로젝트가 아직 시작되지 않았습니다.

👉 /cm-init 을 실행하여 프로젝트를 시작하세요.
   사용법이 궁금하면 /cm-guide 를 실행하세요.
```

### 2. DASHBOARD.md 읽기

`01_init/DASHBOARD.md`를 읽어서 각 단계의 상태를 파악한다.

### 3. 현재 상태 판단

각 단계의 상태를 순서대로 확인:
- ✅ 완료: 다음 단계로
- 🔄 진행 중: 해당 단계 작업 중
- ⏳ 대기: 아직 시작 안 함

가장 먼저 나오는 비완료 단계가 현재 단계이다.

### 4. FEEDBACK.md 확인

현재 단계에 FEEDBACK.md가 있는지 확인:
- FEEDBACK.md가 없고, 산출물이 생성되어 있으면 → "피드백 작성 필요"
- FEEDBACK.md에 ✅가 있으면 → "다음 단계 진행 가능"
- FEEDBACK.md에 🔄가 있으면 → "수정 후 재검토 필요"

### 5. 상태 보고

아래 형식으로 출력:

```
📊 프로젝트 현황

현재 단계: {단계번호}_{단계명} ({상태})
완료 단계: 01 ✅ → 02 ✅ → 03 ✅ → 04 🔄 → ...

{상황별 안내 메시지}

👉 다음 할 일: {구체적인 다음 액션}
   실행할 스킬: /{스킬명}
```

### 상황별 안내 메시지

**산출물 미생성 시:**
```
이 단계의 산출물이 아직 생성되지 않았습니다.
👉 /{해당 CM 스킬} 을 실행하세요.
```

**산출물 생성 완료, 피드백 미작성 시:**
```
산출물이 생성되었습니다. 검토가 필요합니다.
👉 {단계 폴더}/ 의 산출물을 검토해주세요.
   검토 후 {단계 폴더}/FEEDBACK.md를 작성해주세요.
   작성 완료 후 /cm-feedback 을 실행하세요.
```

**피드백 승인 완료 시:**
```
현재 단계가 승인되었습니다.
👉 다음 단계 /{다음 CM 스킬} 을 실행하세요.
```

**사용자 직접 작성 단계(03, 04) 시:**
```
이 단계는 사용자가 직접 작성하는 단계입니다.
👉 {단계 폴더}/{파일명}을 작성해주세요.
   작성 완료 후 /{해당 CM 스킬} 을 실행하세요.
```
